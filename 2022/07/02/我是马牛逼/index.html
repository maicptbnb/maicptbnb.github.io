##帝王

清除：hexo clean

hexo g

开始：hexo s

部署：hexo d

换主题 ： git clone 网站.git themes/butterfly

git提交或克隆报错fatal: unable to access ‘https://github.com/tata20191003/autowrite.git/‘: Failed to connec

1.问题原因
报错信息：

1：fatal: unable to access ‘https://github.com/xxx/autowrite.git/':
2：OpenSSL SSL_read: Connection was reset, errno 10054

又或者

1：fatal: unable to access ‘https://github.com/xxx/autowrite.git/':
2：Failed to connect to github.com port 443: Timed out

因为git在拉取或者提交项目时，中间会有git的http和https代理，但是我们本地环境本身就有SSL协议了，所以取消git的https代理即可，不行再取消http的代理。

后续
原因还有一个，当前代理网速过慢，所以偶尔会成功，偶尔失败。

2.解决方案
1.在项目文件夹的命令行窗口执行下面代码，然后再git commit 或git clone
取消git本身的https代理，使用自己本机的代理，如果没有的话，其实默认还是用git的

1//取消http代理
git config –global –unset http.proxy
2//取消https代理
git config –global –unset https.proxy

2.科学上网（vpn）
这样就能提高服务器连接速度，能从根本解决 time out 443问题
